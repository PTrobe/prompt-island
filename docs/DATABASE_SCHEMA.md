# Database Schema & Data Models

## 1. Overview
The "Prompt Island" backend utilizes a hybrid database approach:
1. **Relational Database (SQLite/PostgreSQL):** Manages the strict game state, agent status, raw chat logs, and voting history. We will use `SQLAlchemy` as the ORM.
2. **Vector Database (ChromaDB):** Manages the unstructured, semantic "Episodic Memories" of the agents for RAG retrieval.

---

## 2. Relational Database Models (SQLAlchemy / Pydantic)

### Model: GameState
Tracks the overarching progression of the reality show.
* `id` (Integer, Primary Key)
* `current_day` (Integer, default=1)
* `current_phase` (String) - e.g., 'morning_chat', 'challenge', 'voting', 'night_consolidation'
* `is_active` (Boolean) - True if the simulation is currently running.

### Model: Agent
Tracks the identity and survival status of each contestant.
* `agent_id` (String, Primary Key) - e.g., 'agent_01_machiavelli'
* `display_name` (String) - The generic human name assigned (e.g., 'Alex')
* `is_eliminated` (Boolean, default=False)
* `eliminated_on_day` (Integer, nullable)

### Model: ChatLog
The immutable, chronological record of everything said and thought. The Frontend/Stream reads from this table.
* `message_id` (Integer, Primary Key)
* `timestamp` (DateTime)
* `day_number` (Integer)
* `phase` (String)
* `agent_id` (String, Foreign Key) - The speaker.
* `action_type` (String) - 'speak_public', 'speak_private', 'system_event'
* `target_agent_id` (String, nullable) - For DMs.
* `message` (Text) - What was actually said out loud.
* `inner_thought` (Text) - The hidden reasoning (vital for the stream UI).

### Model: VoteHistory
Tracks the official votes cast during Tribal Council.
* `vote_id` (Integer, Primary Key)
* `day_number` (Integer)
* `voter_agent_id` (String, Foreign Key)
* `target_agent_id` (String, Foreign Key) - Who they voted to eliminate.
* `reason` (Text) - The justification generated by the LLM.

## 3. Vector Database Model (ChromaDB / Pydantic)

This defines the structure of the long-term memory injected into the RAG pipeline. Claude Code must enforce this schema when embedding the nightly summaries.

```python
from pydantic import BaseModel, Field
from typing import Optional, List

class EpisodicMemory(BaseModel):
    """
    Represents a summarized memory from a specific agent's perspective.
    Stored in ChromaDB for semantic retrieval.
    """
    memory_id: str = Field(description="Unique identifier for the memory chunk.")
    agent_id: str = Field(description="The owner of this memory. Crucial for filtering so agents don't read each other's minds.")
    day_number: int = Field(description="The game day this memory refers to.")
    memory_category: str = Field(description="Must be: 'alliance', 'betrayal', 'challenge_result', or 'general_observation'.")
    content: str = Field(description="The summarized memory text (e.g., 'Player B lied to me today about the vote.').")
    embedding: Optional[List[float]] = Field(default=None, description="The vector representation generated by an embedding model (e.g., text-embedding-3-small).")

## 4. Database Rules for Claude Code

Read-Only Context: When the Agent Controller pulls chat history for the prompt context window, it must query the ChatLog table, filtering by day_number and phase.

Foreign Key Constraints: If an agent is eliminated (is_eliminated = True), they can no longer be inserted into the ChatLog as a speaker unless the action_type is an explicit "final words" event.
